# 混合检索笔记

## 稠密 vs 稀疏向量

| 特性 | 稠密向量 | 稀疏向量 |
|------|----------|----------|
| 生成方式 | BGE-M3 embedding | 类似学习版 BM25 |
| 维度 | 固定 1024 维，每维都有值 | 词表大小，大部分是 0 |
| 匹配方式 | 语义相似 | 关键词精确 |
| 优势 | "机器学习" 能匹配 "ML" | "Python" 精准命中 |

## 什么时候用

- **稠密**：语义理解、同义词、模糊查询、用户表述和文档用词不一致
- **稀疏**：精确关键词、专有名词、代码、低频专业术语

实际做法：不二选一，混合检索效果最好。

## 混合检索流程

```
查询 → BGE-M3 编码 → 稠密向量 → Milvus 检索 → 结果 A ─┐
                   → 稀疏向量 → Milvus 检索 → 结果 B ─┼→ RRF 排名融合
                                                      ↓
                            子块 → 父块去重 → Reranker 精排 → Top-M
```

## 两路结果融合

系统采用 RRF（Reciprocal Rank Fusion，倒数排名融合）融合 dense/sparse 两路召回结果。RRF 不直接比较 dense_score 和 sparse_score 的原始大小，而是根据结果在各路检索中的排名计算融合分。

公式：

```text
rrf_score(doc) = 1 / (k + dense_rank) + 1 / (k + sparse_rank)
```

如果某个文档只出现在一路结果中，就只计算该路排名分。`k` 通常取 60，用于平滑排名差距。

**示例**：

```text
稠密 top-3: [A(第1), B(第2), C(第3)]
稀疏 top-3: [B(第1), D(第2), A(第3)]

RRF 融合后：
B = 1/(60+2) + 1/(60+1)  ← 两路都靠前
A = 1/(60+1) + 1/(60+3)  ← 两路都命中
C = 1/(60+3)
D = 1/(60+2)

最终排序通常会优先保留两路都认可、且排名靠前的结果。
```

两路不一致是好事——说明互补了。

RRF 相比固定加权的好处是：dense 和 sparse 的分数分布不一定可比，直接相加容易被某一路分数尺度带偏；RRF 只看排名，更稳，也不需要额外训练模型。

## 关键参数

| 参数 | 值 | 作用 |
|------|-----|------|
| RRF k | 60 | 平滑排名差距，避免过度放大第一名 |
| RETRIEVAL_K | 5 | 初步召回数量 |
| CANDIDATE_M | 2 | 最终返回数量 |
